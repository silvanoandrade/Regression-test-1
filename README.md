QA Project – Map Functionality Testing

Project Description

This project documents the Quality Assurance (QA) analysis and testing process performed on an interactive map feature within an application. The main objective was to verify the correct behavior of user interactions with the map, identify defects, and document issues through structured test cases and bug reports.

During the sprint, several functional test cases were designed and executed to evaluate behaviors such as map navigation, interaction with visual elements, and the system’s response to different user actions.

Testing Objectives

Validate that the main map functionalities work as expected.
Identify defects or unexpected behaviors.
Document issues with enough detail to reproduce them.
Prioritize bugs based on their impact on the user experience.

Testing Scope

The tests focused on key functionalities of the interactive map, including:
Scrolling and navigation within the map.
Map zoom functionality.
Interaction with map titles or areas.
Behavior of clickable elements.
Correct display of map regions.
System response to different user actions.

Test Cases

A set of structured test cases was created, including the following elements:

Test Case ID
Test Case Name
Preconditions
Execution Steps
Detailed step descriptions

Expected Result

Test Status (Pass / Fail)
Related bug reference (if applicable)
These test cases allow systematic validation of each feature and ensure traceability between executed tests and identified defects.

Bug Reporting

When a defect was identified, a bug report was created including:

Bug ID
Bug title
Steps to reproduce
Expected result
Actual result
Bug priority

The priority levels used were:

Blocker – prevents the feature from being used.
Critical – severely affects system functionality.
Major – impacts functionality but allows continued use.
Minor – small functional issue.
Trivial – cosmetic or low-impact issue.
    
QA Process Applied

Analysis of the map functionality.
Test case design.
Manual test execution.
Identification of defects.
Bug documentation.
Issue prioritization.

Results

The testing process allowed us to:

Validate multiple map functionalities.
Identify interaction and behavior issues.
Generate clear bug reports to support the development team in fixing the defects.
